## 一、软件搭建

### 1.1 环境要求

| 组件 | 版本建议 | 说明 |
|------|----------|------|
| **JDK** | **11** 或以上 | 与 `pom.xml` 中编译目标一致 |
| **Apache Maven** | 3.6+ | 编译、打包、运行、测试 |
| **浏览器** | Chrome / Edge / Firefox | 访问 Web 界面 |
| **Tomcat**（可选） | 9.x / 10.x | 仅独立部署 WAR 时需要 |

环境检查：

```bash
java -version    # 应显示 11 或更高
mvn -version
```

若未安装 JDK，请安装 JDK 11 并设置 `JAVA_HOME`（macOS 示例：`brew install openjdk@11`）。

### 1.2 获取与目录结构

将项目解压或克隆到本地后，进入项目根目录，例如：

```bash
cd /Users/dingyi.zhang/Desktop/软工
```

标准 Maven 布局：

- `src/main/java` — Java 业务与 Web 层代码  
- `src/main/webapp` — JSP、静态资源、`data/` 运行时数据  
- `src/main/resources` — 国际化、小助手配置等  
- `test/java` — JUnit 5 单元测试  
- `pom.xml` — 依赖与构建配置  

### 1.3 编译与打包

在项目根目录执行：

```bash
# 清理、编译并生成 WAR
mvn clean package

# 可选：运行单元测试
mvn test
```

| 产物 | 路径 |
|------|------|
| 可部署 WAR | `target/ta-recruitment.war` |
| 编译结果 | `target/classes/` |

**依赖技术**：Servlet 4.0、JSP、JSTL、Gson、Apache PDFBox、Apache POI。

## 二、配置说明

### 2.1 数据目录（系统运行必需）

启动时由 `AppListener` 自动将数据目录设为 **`{Web 应用根目录}/data`**，无需手工建库。

| 部署方式 | `data/` 位置 |
|----------|--------------|
| `mvn tomcat7:run` 开发运行 | `src/main/webapp/data/` |
| Tomcat 部署 `ta-recruitment.war` | `webapps/ta-recruitment/data/` |

| 文件 / 目录 | 用途 |
|-------------|------|
| `applicants.json` | 应聘者账号与资料 |
| `module_organisers.json` | 课程组织者 |
| `admins.json` | 管理员 |
| `jobs.json` | 岗位信息 |
| `applications.json` | 申请与录用状态 |
| `messages.json`、`dm_read_states.json` | 站内信、已读状态 |
| `forum_threads.json`、`forum_replies.json` | 论坛 |
| `friend_links.json`、`friend_requests.json` | 好友关系 |
| `data/resumes/` | 简历文本文件 |
| `embeddings.json`（可选） | 语义匹配向量缓存 |

用户密码以 **SHA-256** 哈希存储。演示数据已随项目提供；请勿将含真实隐私的生产数据提交至公开仓库。

### 2.2 智能小助手（可选）

默认配置文件：`src/main/resources/assistant.properties`。

| 方式 | 说明 |
|------|------|
| 环境变量 | `KIMI_API_KEY`、`QWEN_API_KEY`、`OPENAI_API_KEY` 等（**优先**） |
| 外部文件 | 环境变量 `ASSISTANT_PROPERTIES_PATH`，或 JVM 参数 `-Dassistant.properties.path=绝对路径` |

**未配置 API Key 时**：招聘、申请、论坛等核心功能正常；仅小助手页面提示不可用。

| 配置项 | 作用 | 默认 |
|--------|------|------|
| `assistant.default.provider` | 模型提供方 kimi / qwen / openai | 按已配置密钥自动选择 |
| `assistant.strict.scope` | 限制提问与招聘无关话题 | 开启 |
| `assistant.monthly.free.quota` | 每用户每月免费对话次数 | 30 |
| `assistant.topup.code` | 兑换码充值（未配微信时） | 空则禁用 |
| `assistant.pay.wechat.*` | 微信 Native 扫码充值 | 见 properties 内注释 |
| `match.semantic.enabled` | 启用 embeddings 语义匹配 | **false** |

### 2.3 界面语言

通过 Cookie `ui_lang` 与 `LocaleFilter` 切换；页眉按钮或 URL 参数 `?lang=zh`、`?lang=en`。

### 2.4 端口与访问路径

| 运行方式 | 访问地址 | 上下文 |
|----------|----------|--------|
| Maven 内嵌 Tomcat | `http://localhost:8082/` | `/` |
| 独立 Tomcat + WAR | `http://localhost:8080/ta-recruitment/` | `/ta-recruitment/` |

默认端口定义于 `pom.xml` 的 `maven.tomcat.port`（当前 **8082**）。端口冲突时：

```bash
mvn tomcat7:run -Dmaven.tomcat.port=8083
```

## 三、运行说明

### 3.1 推荐方式：Maven 内嵌 Tomcat

无需单独安装 Tomcat，适合开发与验收演示。

```bash
cd /Users/dingyi.zhang/Desktop/软工    # 改为你的实际路径

mvn clean package                      # 首次或代码变更后执行
mvn tomcat7:run                        # 保持终端运行，Ctrl+C 停止
```

浏览器访问：**http://localhost:8082/**

### 3.2 备选方式：独立 Tomcat 部署

1. 安装 Tomcat（步骤见 `docs/安装Tomcat.md`）。  
2. 复制 `target/ta-recruitment.war` 到 Tomcat 的 `webapps/`。  
3. 执行 `bin/startup.sh`（Windows：`startup.bat`）。  
4. 访问：**http://localhost:8080/ta-recruitment/**。

更新版本时建议：停止服务 → 删除旧 `webapps/ta-recruitment*` 及 `work/Catalina/localhost/ta-recruitment/` → 部署新 WAR → 重启。

### 3.6 常见问题

| 现象 | 解决方法 |
|------|----------|
| 8082 端口被占用 | `mvn tomcat7:run -Dmaven.tomcat.port=8083` |
| 找不到 `mvn` 命令 | 安装 Maven 并加入系统 PATH |
| 独立 Tomcat 404 | 确认 WAR 已解压，URL 包含 `/ta-recruitment/` |
| 页面 500 或行为异常 | 清理 Tomcat `work/` 下本应用缓存后重新部署 |
| 小助手无法对话 | 检查 API Key；未配置时属正常，不影响招聘主流程 |
