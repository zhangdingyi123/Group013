## 2. 测试环境与前置条件

### 2.1 环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10+ / macOS |
| JDK | **11+** |
| Maven | 3.6+ |
| 运行 | `mvn tomcat7:run` → `http://localhost:8082/`；或独立 Tomcat 9 部署 WAR |
| 浏览器 | Chrome（推荐） |
| 持久化 | `data/*.json`、`data/resumes/` |

### 2.2 启动

```bash
cd <项目根目录>
mvn clean package
mvn tomcat7:run
```

### 2.3 演示账号

| 角色 | 登录路径 | 邮箱 | 密码 |
|------|----------|------|------|
| TA | `/ta/auth` | `liuchen@bupt-demo.edu.cn` | `demo123` |
| MO | `/mo/auth` | `li.prof@bupt-demo.edu.cn` | `demo123` |
| Admin | `/admin/auth` | `admin@bupt-demo.edu.cn` | `admin123` |

> 独立 Tomcat 部署时路径前加 `/ta-recruitment` 等实际上下文。

**优先级**：P0＝演示阻塞；P1＝核心闭环；P2＝增强/边界。

### 2.4 智能小助手

项目部署环境**已配置**大模型 API（Kimi / Qwen / OpenAI 等，见 `AssistantConfig` 加载的配置）。手工测试小助手相关用例时，无需再单独配 Key。
