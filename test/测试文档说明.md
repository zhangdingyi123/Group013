## 2. 测试策略

系统采用 **「单元测试 + 手工黑盒」** 双层策略：

| 层级 | 覆盖重点 | 执行命令 / 方式 |
|------|----------|-----------------|
| **单元测试** | 密码哈希、技能抽取、匹配算法、向量相似度、小助手范围守卫、登录回跳安全、岗位英文校验 | `mvn test`（**不依赖 Tomcat**，当前 **30/30 通过**） |
| **手工黑盒** | Servlet 会话、JSP 页面、文件上传、论坛/私信、全流程招聘、管理员负荷、多语言、小助手真实对话 | 浏览器 + `mvn tomcat7:run` |

**分工原则**：算法与纯逻辑用 JUnit 保证可重复；会话、页面、JSON 持久化、上传与第三方 API 对话用手工验证。

**未覆盖说明**：PRD 中的「岗位收藏」在 `src/main` 中未实现，暂无对应用例。

## 6. 测试执行

### 6.1 自动化

```bash
cd <项目根目录>
mvn test
```

**当前结果**：30 tests，0 failures，0 errors。

### 6.2 手工（发布前冒烟）

1. 启动：`mvn clean package` → `mvn tomcat7:run`
2. **P0 必测**（约 12 条）：首页、三角色登录、未登录拦截、岗位申请与重复申请、简历上传、管理员页、小助手范围内提问
3. 抽查 `/forum`、`/assistant` 无 500

### 6.3 完整回归

- 执行全部 **P0 + P1**（约 **55** 条）
- 对照 `test/测试用例说明.md` 第 3 节覆盖矩阵，每功能域至少 1 条通过

### 6.4 验收清单（10 项）

- [ ] 系统可启动并访问首页
- [ ] Admin / MO / TA 可登录
- [ ] 未登录无法访问受保护页
- [ ] 岗位列表、申请、重复申请拦截正常
- [ ] 个人中心查看与修改正常
- [ ] 简历上传、下载、文本抽取正常
- [ ] 小助手可对话且范围外拒答
- [ ] 论坛可浏览且登录后可发帖
- [ ] 重启后 `data/*.json` 数据仍在
- [ ] 主流程无 HTTP 500
