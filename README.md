# TA 招聘系统

面向北邮国际学院助教（TA）招聘的简易招聘系统原型，用于课程模块负责人及学校各类活动的助教招募。

## 其他强制性要求

**软件开发形式二选一：**
- **a) 独立的 Java 桌面应用程序**
- b) 轻量级的 Java Servlet/JSP Web 应用程序

**数据存储：** 所有输入和输出数据均需存储为简单的文本文件格式（如 .txt、CSV、JSON 或 XML），**禁止使用数据库**。

**框架限制：** **禁止使用 Spring Boot** 等高级框架，仅使用 JDK 与课程要求的轻量依赖（如 JSON 解析），以聚焦软件工程核心原理。

**遵循上述限制的核心目的：**
1. **聚焦软件工程核心原理** — 避免将精力耗费在框架或编程语言的复杂特性上；引入 Spring Boot 等高级工具或进行数据库集成，会使开发重心偏向框架专属特性和配置，偏离本课程的核心学习目标。
2. **系统兼容与互操作** — 保证系统与现有数据兼容，且能与遗留系统实现互操作。
3. **公平性** — 保证所有具备必备前置知识和技能的学生都能在公平的条件下完成项目。

**工作内容：** 运用敏捷方法，完成软件的详细需求定义、设计、开发和测试。在满足客户核心需求、清晰界定项目范围的前提下，团队可自主设计系统架构。

---

**本项目符合性：** 同时提供 **(a) 独立 Java 桌面应用程序**（Swing）与 **(b) 轻量级 Java Servlet/JSP Web 应用程序**（嵌入式 Tomcat），数据仅使用 **JSON 文本文件** 存储；**未使用数据库**，**未使用 Spring Boot**。

## 运行方式概览

| 方式 | 说明 | 入口类 |
|------|------|--------|
| **b) Servlet/JSP Web（推荐）** | 浏览器访问，登录/注册页已优化 | `com.bupt.ta.TomcatMain` |
| a) 独立桌面应用 | 弹出窗口，无需浏览器 | `com.bupt.ta.DesktopMain` |
| 前后端分离（纯 API + 静态页） | 启动 HttpServer 后浏览器访问 | `com.bupt.ta.ServerMain` |
| 控制台版 | 纯命令行 | `com.bupt.ta.Main` |

## 架构说明（b) Servlet/JSP Web）

- **容器**：嵌入式 Tomcat 9（轻量级，非 Spring Boot）。
- **技术**：Servlet 处理登录/注册/工作台，JSP 渲染页面，Session 保持登录状态。
- **登录/注册页**：独立页面、Tab 切换登录/注册、错误提示、居中卡片式布局。
- **数据**：与桌面版、控制台版共用 `data/` 下 JSON 文件。

## 需求实现

- **独立 Java 应用程序**（前后端分离：后端 Java + 前端静态页，无 Servlet/JSP 框架）。
- **仅使用文本文件存储数据**：数据存放在 `data/` 目录下的 JSON 文件中（不使用数据库）。
- **核心功能**：
  - **助教（TA）**：创建个人资料、上传简历（路径）、查找职位、申请职位、查看申请状态。
  - **课程负责人（MO）**：发布职位、为职位选择申请人。
  - **管理员（Admin）**：查看助教整体工作量（每位 TA 已分配职位数量）。

## 编译与运行

```bash
# 编译
mvn compile
```

**1. 轻量级 Servlet/JSP Web 应用（推荐）**

```bash
mvn exec:java -Dexec.mainClass=com.bupt.ta.TomcatMain
```

或在 IDE 中运行 **`com.bupt.ta.TomcatMain`**。启动后在浏览器打开 **http://localhost:8080/** ，使用优化后的登录/注册页与工作台。

**2. 独立桌面应用**

```bash
mvn exec:java -Dexec.mainClass=com.bupt.ta.DesktopMain
```

**3. 前后端分离（HttpServer + 静态前端）**

```bash
mvn exec:java -q
```

默认启动 `ServerMain`，浏览器访问 http://localhost:8080/ ，API 根地址为 http://localhost:8080/api 。

**4. 控制台版**：在 IDE 中运行 **`com.bupt.ta.Main`**。

若提示端口 8080 已被占用，请先关闭其他占用该端口的程序（如已运行的 ServerMain 或 Tomcat），再启动 Servlet/JSP 版。

## 数据存储

所有数据存放在 `data/` 目录下（首次运行时自动创建）：

- `applicants.json` — 助教申请人资料
- `jobs.json` — 课程负责人发布的职位
- `applications.json` — 助教申请记录及状态（待处理/已录用/已拒绝）
- `module_organisers.json` — 课程负责人账号

## 使用说明（前端）

1. **助教（TA）**：首页选择「助教」→ 登录或注册 → 工作台内维护资料/简历路径/技能、查看可申请职位、提交申请、查看申请状态。
2. **课程负责人（MO）**：首页选择「课程负责人」→ 登录或注册 → 发布职位、查看我的职位、按职位加载申请并录用。
3. **管理员（Admin）**：首页选择「管理员」→ 查看 TA 工作量表（每人已分配职位数）。

可选后续扩展（按规格说明）：技能匹配、缺失技能提示、工作量均衡等。
